ShortcutBlocker — 让游戏不再被系统快捷键抢走按键

一个蛋疼的场景你打开一个 FPS 游戏，按住 Ctrl 蹲下，同时按下 Space 准备跳起来爆头。结果——输入法切换出来了，游戏里的角色纹丝不动。 更烦人的是，不只是 Ctrl+Space。Ctrl+Shift 切输入法、Win+Space 切输入法、Alt+Tab… 这些 Windows 系统快捷键在游戏里一个比一个碍事。 市面上不是没有拦截工具，但绝大多数都一个粗暴逻辑：检测到快捷键，直接 return 1 把按键吞掉。输入法是不切了，但游戏也收不到按键了——按住 Ctrl 的时候按 Space，游戏根本不认为你按了跳跃。 这就很离谱：我要的是输入法别切，不是游戏也别收到。 解决思路ShortcutBlocker 的做法分两层： 阻断系统层：用 Windows 全局低级键盘钩子（WH_KEYBOARD_LL）在按键到达 IME 之前拦截，return 1 让 IME 完全不知道这个按键的存在 透传应用层：用 PostMessage 把按键消息直接塞进游戏窗口的消息队列，不走系统键盘管道，IME 碰不到，游戏的消息循环能正常收到 为什么不用 SendInput？第一次实现我试了 SendInput 重新注入按键，但这条路走不通： SendInput 注入的事件还是会走系统键盘管道，而 Ctrl 是物理按下的，系统又把它当成 Ctrl+Space 交给 IME 处理了 部分游戏不认 SendInput 注入的事件 改用 PostMessage 之后，消息直接投递到目标窗口的队列里，绕过了整个系统输入管道，IME 完全没有机会劫持。
